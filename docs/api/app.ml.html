app.ml package

app.ml.classification_utils module

This is a simple classification service. It accepts an url of an image and returns the top-5 classification labels and scores.

app.ml.classification_utils.classify_image(model_id: str, img_id: str) → list[source]

Classifies an image using the specified pre-trained model.

This function feeds the specified image into a pre-trained model and returns the top-5 classification results.

Parameters:

  • model_id (str) – The identifier of the pre-trained model to be used for classification.

  • img_id (str) – The identifier (filename) of the image to be classified.

Returns:

A list containing the top-5 classification results, where each item is a tuple of (label_name: str, confidence_score: float).

Return type:

list of tuple

app.ml.classification_utils.fetch_image(image_id: str) → Image[source]

Retrieves an image from the edited, upload, or dataset folder.

This function attempts to fetch an image using the provided image ID. - It first checks if an edited version of the image exists in the ‘edited’ folder. - If not found, it looks in the upload folder. - If still not found, it retrieves the image from the default dataset folder.

Parameters:

image_id (str) – The filename or identifier of the image to be retrieved.

Returns:

The opened image file as a PIL Image object.

Return type:

Image.Image

Retrieves the ImageNet class labels.

Returns:

A list of ImageNet labels where each index represents a class.

list of str

app.ml.classification_utils.get_model(model_id: str)[source]

Loads a pre-trained model from the configuration.

This function imports a specified model from torchvision.models using the given model ID. The model is pre-downloaded to prevent unnecessary waiting during classification.

Parameters:

model_id (str) – The identifier of the model to be loaded (must be in conf.models).

Returns:

The loaded PyTorch model with default pretrained weights.

Return type:

torch.nn.Module

Raises:

ImportError – If the specified model is not found.
